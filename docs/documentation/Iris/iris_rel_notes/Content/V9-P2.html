New Features and Enhancements in V9 Preview2

In this release, the following features have been introduced to provide an enhanced HCL Iris experience.

 

  • Support for KSLint
  • Workflow Service support in the Data and Services Panel
  • Project Settings Enhancements
  • Jasmine Test Automation Enhancements
  • Resolve Action Conflicts in Bulk
  • Custom widget support
  • ControllerActions Files In Gitignore
  • Capture Image Metadata
  • Widget Support for the CursorType Property
  • Convert Integer to Number through Android FFI
  • Deprecated Support for Windows 8

 

Support for KSLint

HCL Iris V9 provides support for a new linter, KSLint. KSLint identifies errors in code and also any deviations from basic JavaScript code guidelines. KSLint allows developers to configure default rules and add new custom rules.

For more information on KSLint, click here.

Workflow Service support in the Data and Services Panel

From HCL Iris V9, you can configure, edit, and view Back-end workflows associated with a Service through the Data and Services Panel.

For more information on the Workflow support, click here.

Project Settings Enhancements

The Project Settings section is enhanced to port a few features from HCL Iris Classic into HCL Iris. New settings introduced are Android Wear Settings, Platform Settings for iOS, and General Tab in the Adaptive Web Settings.

For more information about this feature, click here.

Jasmine Test Automation Enhancements

From HCL Iris V9, the following enhancements have been made to the Jasmine Test Framework:

  • Ability to modify the port on which the HCL Automator runs. For more information about this feature, click here.
  • Ability to edit a recorded test case manually through code with the Edit Test Case Manually option. For more information on this feature, click here.

Resolve Action Conflicts in Bulk

The Resolve Action Conflicts feature in HCL Iris V9, helps a developer to resolve any conflicts in the actions of a widget or form when a project is imported from HCL Iris Classic to HCL Iris.
This feature has been introduced to support the porting of features from HCL Iris Classic into HCL Iris.

For more information about this feature, click here.

Custom widget support

In earlier versions of HCL Iris, importing custom widgets into SPA and Desktop Web applications was not possible. From HCL Iris V9, developers can import custom widgets into SPA and Desktop Web applications. Developers can also add custom web widgets as a component into an application.

For more information about this feature, click here.

ControllerActions Files In Gitignore

For a form created using the Reference Architecture, HCL Iris creates a controlleractions.js file corresponding to the form. When the project is hosted on GIT, during the commit process, as HCL Iris creates the controlleractions.js file every time Iris restarts, GIT commits will result in conflicts. To avoid these conflicts, the controlleractions.js file commit process is modified, and now the file will automatically be added to the .gitignore folder.

For more information about this feature, click here.

capture Image Metadata

From HCL Iris V9, when an image is captured using a Camera widget for Android and iOS applications, you can retrieve information such as height, width, EXIF version, etc. of the captured image.

For more information about this feature, click here.

Widget Support for CursorType Property

From HCL Iris V9, you can assign the mouse pointer type to a widget by navigating to the Hover Skin tab in the Skin tab in the Properties panel of HCL Iris.

For more information about this feature, click here.

Convert Integer to Number through Android FFI

In earlier versions of HCL Iris, when an FFI code returns an integer value, it was passed as an opaque object. From HCL Iris V9, all Integer values that are returned from an FFI code are passed in the Number type to the JavaScript code.

Deprecated Support for Windows 8

From HCL Iris V9, support for Windows 8 devices has been deprecated.
