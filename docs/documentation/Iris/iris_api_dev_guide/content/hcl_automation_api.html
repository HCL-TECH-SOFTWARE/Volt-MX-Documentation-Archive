VoltMX Iris Automation Framework

VoltMX Iris Automation Framework provides you a more convenient way to test your application across various platforms. The framework follows the Write Once, Run Anywhere methodology, so that different automation efforts put for each platform for the same application can be reused. JavaScript is used for VoltMX Iris Automation, and you do not need to have expertise on any other native language to use it. It is currently supported on iOS, Android, Windows, and DesktopWeb, SPA platforms.

JavaScript Testing Framework – Jasmine

Jasmine is a behavior-driven development framework for testing JavaScript code. The framework does not depend on any other JavaScript frameworks. Furthermore, the framework has a clear and obvious syntax, thus enabling you to easily write tests.

VoltMX Iris Automation APIs: Supported Items

VoltMX Iris Supports automation for both MVC and non-MVC projects. Automation API supports the following widgets:

  • All views of all widgets

  • Platform-specific widgets

  • Masters and User widgets (components)

  • Deprecated widgets - Box, Form, ScrollBox, etc.

In addition, Automation API supports the following items:

  • Low-level actions on all supported widgets

  • Touch

  • Scroll

  • Hardware buttons

  • Back button

  • Menu button

  • Text entry

  • Camera capture

  • Map and browser interaction

VoltMX Iris Automation API consists of the hcl.automation Namespace and its following inherent API groups:

  • Widget APIs
  • Low-level Touch and Gesture APIs
  • Miscellaneous Automation APIs
  • Existing VoltMX Iris APIs

The VoltMX Iris Automation API uses the hcl.Automation namespace and the following API elements:

Methods Description
[`hcl.automation.alert.click`](hcl.automation_namespace.htm#alert.click) Clicks the provided button, if visible, for the Alert. If multiple alerts are simultaneously shown, this API clicks the most recent alert displayed on the screen.
[hcl.automation.box.click](hcl.automation_namespace.htm#box.click) Triggers the Box click event on the specified widget, if it is visible and enabled.
[hcl.automation.button.click](hcl.automation_namespace.htm#button.click) Triggers the Button click event on the specified widget, if it is visible and enabled.
[hcl.automation.calendar.selectDate](hcl.automation_namespace.htm#calendar.selectDate) Triggers the Calendar click event on the specified widget, if it is visible and enabled.
[hcl.automation.checkboxgroup.click](hcl.automation_namespace.htm#checkboxgroup.click) Triggers the CheckBoxGroup click event on the specified widget, if it is visible and enabled.
[hcl.automation.flexcontainer.click](hcl.automation_namespace.htm#flexcontainer.click) Triggers the FlexContainer click event on the specified widget, if it is visible and enabled.
[hcl.automation.radiobuttongroup.click](hcl.automation_namespace.htm#radiobuttongroup.click) Triggers the RadioButtonGroup click event on the specified widget, if it is visible and enabled.
[hcl.automation.segmentedui.click](hcl.automation_namespace.htm#segmentedui.getItem) Triggers the SegmentedUI click event on the specified widget, if it is visible and enabled.
[hcl.automation.segmentedui.pull](hcl.automation_namespace.htm#segmentedui.pull) Triggers the onPull event on the SegmentedUI, if it is set.
[hcl.automation.segmentedui.push](hcl.automation_namespace.htm#segmentedui.push) Triggers the onPush event on the SegmentedUI, if it is set.
[hcl.automation.segmentedui.scrollToRow](hcl.automation_namespace.htm#segmentedui.scrollToRow) Triggers the segment to scroll to the row specified by the index of the specified widget, if it is visible and enabled.
[hcl.automation.segmentedui.scrollToWidget](hcl.automation_namespace.htm#ScrollToWidget) Scrolls to ensure that the widget appears in view. It is an awaitable API.
[hcl.automation.slider.slide](hcl.automation_namespace.htm#slider.slide) Triggers the Slider slide event on the specified widget, if it is visible and enabled.
[hcl.automation.switch.toggle](hcl.automation_namespace.htm#switch.toggle) Toggles the Switch between ON/OFF on the specified widget, if it is visible and enabled.
[hcl.automation.tabpane.click](hcl.automation_namespace.htm#tabpane.click) Clicks the tab with the specified tabID on the TabPane widget, if it is visible and enabled.
[hcl.automation.textbox.enterText](hcl.automation_namespace.htm#textbox.enterText) Enters the specified text into the TextBox, if it is visible and enabled.
[hcl.automation.textarea.enterText](hcl.automation_namespace.htm#textarea.enterText) Enters the specified text into the TextArea, if it is visible and enabled.
[hcl.automation.widget.touch](hcl.automation_namespace.htm#widget.touch) Triggers the touch event on the specified widget, if it is visible and enabled.
[hcl.automation.playback.wait](hcl.automation_namespace.htm#playback.wait) Introduces a delay time in the playback as specified. It is an awaitable API.
[hcl.automation.playback.waitFor](hcl.automation_namespace.htm#playback.waitFor) Waits for the widget to load completely. It is an awaitable API.
[hcl.automation.device.deviceBack](hcl.automation_namespace.htm#device.deviceBack) Invokes the back action of the device. It is an awaitable API.
[hcl.automation.capture](hcl.automation_namespace.htm#capture) Takes a screenshot of the widget passed. If the widget does not pass, the screenshot captures the entire screen.
