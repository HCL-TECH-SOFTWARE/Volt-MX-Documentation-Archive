Passbook API

Use a passbook to store items like boarding passes, movie tickets, and gift cards. The Passbook API lets you scan your iPhone or iPod touch to check-in for a flight, get into a movie, redeem a coupon, and more.

Important: Ensure that you have the iOS Passbook FFI imported into your project before you use HCL’s Passbook API. The Passbook FFI is mandatory for the steps in this topic to work.

The Passbook API contains the following objects and related methods:

PassLibrary Object

Method Description
[addPassWithCompletionCallback](passlibrary.htm#getAccur) Presents UI to add pass.
[addPassesWithCompletionCallback](passlibrary.htm#getMajor) Presents UI to add multiple passes.
<p>[containsPass](passlibrary.htm#getMinor) </p> Returns whether the user’s pass library contains a pass.
[getPassWithTypeIdentifierAndSerialNumber](passlibrary.htm#getProxi) The proximity value gives a general sense of the relative distance to the beacon.
[getPasses](passlibrary.htm#getProxi2) Returns the passes in the user’s pass library.
[removePass](passlibrary.htm#getrssi() Removes the pass from the user’s pass library.
[replacePassWithPass](passlibrary.htm#replaceP) Replaces a pass in the user’s pass library with the given pass.

 

You can add passes to a pass library using the [addPassWithCompletionCallback](passlibrary.htm#getAccur) or [addPassesWithCompletionCallback](passlibrary.htm#getMajor) functions. You can retrieve the available passes using the [getPasses](passlibrary.htm#getProxi2) or [getPassWithTypeIdentifierAndSerialNumber](passlibrary.htm#getProxi) functions. Use the [replacePassWithPass](passlibrary.htm#replaceP) function to replace a pass with another pass, and the [removePass](passlibrary.htm#getrssi() function to remove a pass.

Pass Object

Method Description
[getAuthenticationToken](pass.htm#getAccur) The token used to authenticate with the web service.
[getLocalizedDescription](pass.htm#getMajor) The localized description of the pass’s kind.
[getLocalizedName](pass.htm#getLocal) Use this property to provide accessibility information for a UI element that represents a pass, such as a cell in a table view.
[getLocalizedValueForKeyForDeviceLocale](pass.htm#getMinor) Returns the localized value for specified field of the pass.
[getOrganizationName](pass.htm#getProxi) The name of the organization that created the pass.
[getPassTypeIdentifier](pass.htm#getProxi2) The pass’s type identifier.
[getPassURL](pass.htm#getrssi() The URL that opens the pass in the Passbook app.
[getSerialNumber](pass.htm#replaceP) A value that uniquely identifies the pass.
[getUserInfo](pass.htm#getUserI) Developer-specific custom data.

You can retrieve the authentication token for a pass using the [getAuthenticationToken](pass.htm#getAccur) function. You can retrieve localized information about the pass using the [getLocalizedDescription](pass.htm#getMajor), [getLocalizedName](pass.htm#getLocal), or [getLocalizedValueForKeyForDeviceLocale](pass.htm#getMinor) functions. Using the [getOrganizationName](pass.htm#getProxi) function, you can retrieve the name of the organization that created the pass. You can retrieve information about the pass using [getPassTypeIdentifier](pass.htm#getProxi2), [getPassURL](pass.htm#getrssi(), or [getSerialNumber](pass.htm#replaceP) functions, and retrieve information about the user using the [getUserInfo](pass.htm#getUserI) function. You can dismiss the add passes view using the [dismissAnimated](com.hcl_objects_addpassesviewcontroller_methods.htm#getAccur) function.

Here is an example for creating a isPassLibraryAvailable with callbacks:

var pass1 = new com.hcl.isPassLibraryAvailable();

Input Parameters

None

Return Values

Boolean - True if available, false if not available.

Platform Availability

Available only on iOS.

Interacting with Passes

The HCLPassLibrary helps you interact with the Passes in iOS devices. All the Passes are contained within a PassLibrary.

The PassLibrary object represents the library of passes, and a Pass object represents an individual pass.

Determine the Availability of a Pass Library

The developer must check the availability of the iOS Pass library in the current device.

Use the following snippet to determine the availability of a pass library:

​​​​​

Check whether a Pass is in the Pass Library

Use the containsPass method to determine whether a pass is in the user pass library.

​​​​​

Access Passes from the Pass Library

Use the getPasses method to get all the passes that a developer’s application is entitled to access.

Read a Pass from the Pass Library

Use the getPassWithTypeIdentifierAndSerialNumber method to read a pass from pass library.

Developer can use methds such as getOrganizationName, getLocalizedDescription of Pass class to access information about the pass.

Use the getLocalizedValueForKeyForDeviceLocale method to access specific field data for a key.

​​​​​

Add a Pass to the Pass Library

AddPassesViewController is a view controller which can be used to display passes and let users add them to their pass library.

Create a pass with pass data file path, use a AddPassesViewController object to add passes to pass library.

​​​​​

Modify a Pass

Application can download a new pass(signed) from server and replace it in the pass library using replacePassWithPass method.

​​​​​

Remove a Pass from the Pass Library

Use the [removePass](passlibrary.htm#getrssi() method of PassLibrary class to remove a pass from pass library.

​​​​​

Open a Pass in the Passbook application

Use [getPassURL](pass.htm#getrssi() property to present the pass in Passbook.
