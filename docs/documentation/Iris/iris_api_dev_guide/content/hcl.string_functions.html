hcl.string Namespace

The hcl.string namespace forms the String API and provides the following API elements.

Functions

The hcl.string namespace provides the following functions.

hcl.string.containsChars

This API verifies if any one of the specified set of characters is available in the given string and returns a boolean value.

Syntax

hcl.string.containsChars (inputstring, characterArray)

Input Parameters

Parameter Description
inputstring [String] - Mandatory Specifies the input string that needs to be verified.
characterArray [Array] - Mandatory Specifies the set of characters that need to be searched within the input string.

 

Example

In this example, hcl.string.containsChars returns true if the input string consists the characters specified in the characterArray. For example, since the input string _“abcd ADV”_ consists the characters specified in the character array, hcl.string.containsChars returns true.

​​​​​

Return Values

Return Value Description
status [Boolean] <p>true - if the given input string contains any one of the characters in the specified list.</p><p>false - if there is an error in the input or if the given input string does not contain any of the specified input characters.</p>

Exceptions

An error is thrown if input type is invalid or follows an unexpected structure.

102-Invalid input error.

Platform Availability

Available on all platforms.

hcl.string.isValidEmail

This API verifies if the input string is a valid email address and returns a boolean value.

Syntax

hcl.string.isValidEmail (inputstring)

Input Parameters

Parameter Description
inputstring [String] - Mandatory Specifies the input string that needs to be verified if it is a valid email address.

 

Example

In this example, hcl.string.isValidEmail returns true only if the input string comprises a valid email address.

​​​​​

Return Values

Return Value Description
status [Boolean] <p>true - if the given input string satisfies the email rules and is a valid email address.</p><p>false - if there is an error in the input or if the given input string does not satisfy email rules.</p>

Rules and Restrictions

The API performs the following validations on the input string:

  • has at least 6 characters.
  • has @.
  • has at least 4 characters after @.
  • has . (dot) followed by at least 2 characters.
  • has at least 1 character before @.

Exceptions

An error is thrown if input type is invalid or follows an unexpected structure.

Platform Availability

Available on all platforms.
