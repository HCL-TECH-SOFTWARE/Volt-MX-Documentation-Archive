Upgrade KMS (Engagement Server) from Pre-7.x to latest HCL Foundry

The following steps help you upgrade KMS from 6.5.x to latest HCL Foundry:

Important: Back up your database schema and hclpns.war file (from server) before proceeding with the upgrade.

Upgrading the KMS manually involves the following four steps:

  • Download the kms zip file
  • Database upgrade
  • War upgrade
  • Configure the properties files

  1. Step 1: Download the kms zip file.

    Download the required 7.x version of HCL Foundry Messaging (kms.zip) from the following location: http://community.hcl.com/downloads/manual.

  2. Step 2: Upgrade the database.

    Unzip kms.zip file to see the .tar file related to your database type. For example, mysql.tar.

    Extract the .tar file to see the following files and folders. [There are .sql scripts are available in the Upgrade folder as well as under the extracted folder as highlighted in the following screenshot:

    Refer the Database Upgrade for KMS section (go to the section based on your database) to run the .sql file in the order mentioned in the documentation.

    Important: Database upgrade scripts are available until 7.1.0.3 in community portal. To upgrade KMS to 7.2 or later, you must raise a support ticket to get the scripts.

  3. Step 3: War Upgrade in Server.

    Note: Back up your files before deleting/overwriting.

    1. Deploy hclpns war in Tomcat Server.

      To deploy HCL Foundry Engagement Services on Tomcat Server, copy the hclpns.war (which is available in the extracted hclpns.zip file) file to the following location: <Tomcat installation directory>/webapps/

    2. Remove hclpns folders from the following two paths and restart Tomcat server:

      <Tomcat installation directory>/work/Catalina/<host>/

      <Tomcat installation directory>/webapps/

    For other web servers or application servers, follow the deployment steps required by the respective server.

  4. Step 4: Configure the properties files.

    1. Extract the hclpns-resources.tar file (which is available in the extracted hclpns.zip file) and place the files in a folder.

    2. Modify the configResource.properties, database.properties and kms-log4j2.xml files based on your existing setup.

    3. Provide the path of these files as a –D param with the param name hclpns.configLocation param.

    4. Refer the Setup HCL Foundry Engagement Services section for more information.

    Important: Currently, the upgrade process remains same for all the versions.
