AppFactory User Guide: Running Iris App > Auto-triggered Jobs

Auto-Triggered Jobs

Auto-Triggered Jobs in AppFactory are tasks that run based on a trigger or a scheduler. The triggers and schedulers are specialized jobs that can be used to automate and run tasks in your AppFactory projects.

Consider the following scenarios:

  • A developer wants to verify whether an app runs after code changes are pushed to the source control.
    The administrator can create a watcher that runs the build and test jobs whenever a change is pushed to the source code.
  • A developer wants to implement a nightly build system.
    The administrator can create a scheduler that runs the build jobs at a specified time.

To create a trigger, you must create a Webhook in the SCM that you use for your AppFactory projects. For more information about creating Webhooks, refer to the following sections.

  • Creating Webhooks in GitHub
  • Creating Webhooks in Bitbucket

After you create a webhook, you can configure auto-triggered jobs on AppFactory. For more information, refer to Configuring Auto-triggered Jobs on AppFactory.

Every project contains a sub-folder called Triggers in the Iris folder. The Triggers folder contains a job called createTrigger (to create auto-triggered jobs), and two sub-folders called watchers and schedulers (to store the created jobs).

Creating Webhooks in GitHub

Prerequisites

Make sure that you have permissions to change the settings of the GitHub repository.

Configuring a Webhook

  1. Open your GitHub repository in a web browser.
  2. Navigate to the Settings tab.

  1. On the left navigation panel of the Settings tab, click Webhooks. The Webhooks page opens.

  1. On the upper-right corner, click Add webhook.

  1. On the confirm password dialog box, type your GitHub password, and then click Confirm password.
    After successful authorization, the Add webhook page opens.

  1. Configure the following parameters in the Add webhook page: * Type the Payload URL (mandatory field) based on the following template:

    <Jenkins URL>/github-webhook/ (last forward slash is mandatory);
    Where **** is `://`, for example - `https://ci.hcl.com` For more information, refer to [Payload URL Format](#PayloadURL).

    After you type the payload URL, GitHub automatically checks the https connection and displays a prompt to disable SSL verification. Ignore the prompt.

*   Content type must be `application/x-www-form-urlencoded`
*   Jenkins verification does not require a **Secret**. Leave the field empty.
*   Select the **Just the push event** option. Only pushes of code to a specific branch trigger the **buildIris** job.
*   The **Active** option should be marked by default.

![](/volt-mx-docs/docs/documentation/Foundry/hcl_appfactory_user_guide/Content/Resources/Images/AT_WebhookParams.png)
  1. After you configure the necessary parameters, click Add Webhook.

  1. After a Webhook is created, the Webhooks page appears.
    Every Webhook displays a message that contains the Payload URL and the supported events (in brackets).
    For example: https://hcl-elb-12345678.us-west-1.amazonaws.com/github-webhook/ (push)

  1. You can Edit or Delete a Webhook by clicking the respective button. For more information about Webhooks, refer to the GitHub Webhook Documentation.

Creating Webhooks in Bitbucket

Prerequisites

Make sure that you have permissions to change the settings of the Bitbucket repository.

To enable triggers in Bitbucket (Stash), you must install the Webhook to Jenkins for Bitbucket add-on. To install the add-on, follow these steps:

  1. Sign-in to your Bitbucket instance as an admin.
  2. Click the admin drop-down list, and then click Add-ons. The Manage add-ons screen appears.
  3. Click Find new add-ons from the left-hand side of the page
  4. Locate Webhook to Jenkins for Bitbucket by using the search option. The results include the app versions that are compatible with your Bitbucket instance.
  5. Click Install to download and install your app.
  6. Click Close.

For more information about add-ons, refer to the Stash Webhook to Jenkins page.

Configuring a Webhook

  1. Open your BitBucket repository in a web browser.
  2. On the left-navigation panel, click Settings.

  1. On the Settings page, in the WORKFLOW section, click Hooks.

  1. On the Hooks page, for the Stash Webhook to Jenkins option, click Enable. The Webhook configurations section opens.

  1. Add the Jenkins URL and the Repo Clone URL.

    The Jenkins URL is Jenkins home page URL.

    The Jenkins URL is the URL of the Jenkins home page. The Repo Clone URL is the URL that is used to clone the repository. The Trigger Jenkins button is used to check whether you can trigger the required Jenkins job. If the trigger is successful, click Enable in the lower-right corner.

  1. To edit the Webhook, click the edit icon. To disable the Webhook, click Disabled.

Configuring Auto-triggered Jobs on AppFactory

To set up an auto-triggered job in AppFactory, follow these steps:

  1. From the folder of your project, navigate to the Triggers sub-folder.

  1. Select the createTrigger job.

  1. Configure the CRON_SCHEDULE parameter based on your preference
    • If you want to schedule a build, type a Cron expression that corresponds to the time at which the build job must be triggered.
      Jobs that are triggered by Cron expressions are stored in the Schedulers folder.
      For information about cron expressions, refer to Cron.
    • If you want to trigger a build based on changes to the SCM, leave the CRON_SCHEDULE parameter blank.
      Jobs that are triggered by the SCM are stored in the Watchers folder.

  1. Configure the remaining parameters, which are the same as the buildIris job.

Every Scheduler contains a cron string in the name to simplify job search.

Every Watcher has a branch name in the name of Job to simplify job search.

Schedule Nightly Builds

You might have a requirement to build applications at night on a regular basis. Nightly builds provide better performance and efficient resource allocation. Nightly builds are also used for solid product building and testing without ongoing development.

To configure a nightly build for your project, create a Scheduler for your project as mentioned earlier (Configuring Auto-triggered Jobs on AppFactory). For the CRON_SCHEDULE parameter, type the Cron expression: 0 0 * * *. The createTriggers job creates the Schedule_0 0 * * * job in the Schedulers folder. This job is launched everyday at 00:00 hrs (12:00 AM).

Reference: Payload URL Format

The format of the Payload URL depends on the type of GIT server. AppFactory supports the following GIT servers:

Server Type

Jenkins URL

GitHub Server

<Jenkins URL>/github-webhook/

Bitbucket Server (Stash)

<Jenkins URL>/bitbucket-hook/
