New Features in V8 SP2

Console

  • Added support for versioning of VoltMX Foundry applications. For more information, refer VoltMX Foundry Application Versioning.

  • Added support for viewing the published applications from the Environment page. For more information, refer Viewing Applications Published to a Runtime Environment.

  • Added support to improve the testing experience for Integration Service. For more information on new testing experience, refer [Integration Service Designer](/volt-mx-docs/docs/documentation/Foundry/voltmx_foundry_user_guide/Content/Services.html#IntSD?TocPath=Features Integration _____1).

  • Added support for testing API proxy connector and importing API Proxy operations from an OpenAPI (Swagger) file. For more information, refer Adding API Proxy.

  • Publish reconfiguration support is extended to all Integration services. For more information, refer Application Reconfiguration.

  • Added support that enables users with “Report Full Access” to run the reports without accessing the cloud.

Offline Objects

  • Integrated offline enabled Objects API access through MVC models.

  • Added support for Service-Driven Objects (SDO) for Offline Objects. For more information, refer [Support for SDO from Offline Objects](/volt-mx-docs/docs/documentation/Foundry/voltmx_foundry_user_guide/Content/Introduction.html#ObjectsServices/Objectservices_Stage3.html#mapping-verbs-for-objects-for-service-driven-objects?TocPath=Features Object_Services Stage_3#_Mapping_Operations_to_Back-end_Methods Mapping_Verbs_and_Methods_to_the_Fields_on_the_Back_End _____4).

  • Added support for Sync Config Policies (Upload/Download). For detailed information, refer Sync Configuration Policies.

  • Added support to download the binary data. For detailed information on downloading binary data, refer Object getBinary.

  • Added support to disable change tracking. For API information, refer Offline Objects API Reference Guide.

  • Added support to mark records for upload. For information on Mark for Upload API, refer Object Mark for Upload.

  • Offline Objects support incremental metadata support. For information on Incremental setup API, refer Object Incremental Setup.

  • Added support for cancelSync API. For API information, refer Offline Objects API Reference Guide.

  • Offline Objects now support upload cache. For information on Upload Cache, refer Configuring Offline Objects Services Server.

  • Offline Objects provide batching support for hierarchical data. For more information, refer Batching Support .

VoltMX Foundry SDK

  • Added binary support for S3 and File System (via NFS drive). For more information, refer Binary APIs.

  • Ability to cache the response of an Integration/Object/Identity/Logic Service is provided.

  • Added support to fetch client app properties configured in App Services.

Identity

  • Added support to publish different identity service configurations to different environments in an account. . If you are accessing identity services directly, refer Publishing of identity services to different environments for additional parameters that needs to be passed.

  • Added support to HTTP message body integrity from VoltMX Foundry. For more information, refer How to enable HTTP Message Body Integrity.

Accounts/Metrics

  • Added Web Application Firewall (WAF) support to VoltMX Foundry products.

  • Added support to optimize data load logic to increase performance.

Integration

  • Enabled the support for testing services in VoltMX Foundry Starter Edition.

  • Added throttling support for Object and Orchestration services.

  • Added support for dynamic loading of SSL certificates. For more information on SSL, refer SSL Certificates.

  • Introduced real-time server monitoring in App Services. For more information, refer Service Monitor.

  • Enabled safe JavaScript in VoltMX Foundry Community Edition.

  • Provision to access Identity scope parameters in Java and JavaScript Integration services. For more information, refer [Integration](/volt-mx-docs/docs/documentation/Foundry/voltmx_foundry_user_guide/Content/Services.html?TocPath=Features Integration _____0).

  • Provision to enable General Data Protection Regulations (GDPR) of EU in App Services. For GDPR settings, refer GDPR Policy.

  • Excel based modifiable CSV data files are exported from storage services.

  • Significant Improvement in publishing performance and added support of zero downtime during publishing.

Installer

  • Offline Sync features are now available with Integration component using Offline Objects. For more information, refer Offline Objects Getting Started Guide. Sync Services is deprecated as a separate component, and will be unselected by default in the Install Components Screen.

  • The new version of Apache Tomcat v8.5.28 is bundled with the installer. The new Tomcat version addresses security concerns highlighted in http://tomcat.apache.org/security-8.html#Fixed_in_Apache_Tomcat_8.5.28.

  • New version of Websphere v9 is supported for remote deployments.

Management

  • Enterprise Appstore for VoltMX Foundry Community edition.

  • Automate JAR generate to do away with JAR commits for Launchpad.

Engagement

  • Added support for cloud push processing optimizations.

  • Importing Events - The new feature enables the admin to perform insert, update/insert, and delete all & insert of events in VoltMX Engagement Server. The feature helps the user to do a bulk import of events (for Cloud version only).

  • Retain Event Registration ID - While importing events, the new feature enable to retain the existing event ID (19 digits) in the system if needed (for Cloud version only).

  • Importing Templates (Push, SMS, email, and Pass) - The new feature enables the admin to perform insert, update/insert, and delete all and insert of templates in VoltMX Engagement Server. The feature helps the user to do a bulk import of templates (for Cloud version only).

  • Role-based Custom Attribute Creation Permission - The user can add a custom user attribute only if the Admin has assigned the following permissions (for Cloud version only):

    • ROLE_CONFIGURATIONS

    • ROLE_CREATE_ATTRIBUTES

  • SQL performance enhancements in Engagement server (cloud and on-prem).

App Factory

  • Added support for Custom Hooks. For more information, refer Running CustomHooks.

  • Added capabilities for better debugging. For more information, refer Troubleshooting.
